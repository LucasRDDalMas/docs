# Docs POC

## Goal

> Build a well-structured Obsidian vault that serves as a living documentation system for the team.

## Context

The team lacks a centralised place to capture decisions, project notes, and reference material.
Obsidian is being evaluated as the solution because it is local-first, Markdown-native, and
supports powerful linking and querying via Dataview.

## Scope

### In Scope

- Vault folder structure
- Core templates (project, meeting, daily note, area, resource)
- Example notes in each category
- Conventions guide

### Out of Scope

- Plugin configuration beyond Dataview
- Sync setup (iCloud / Obsidian Sync decisions deferred)

## Notes

Using PARA as the base organisational system:
- **P**rojects — time-boxed with outcomes
- **A**reas — ongoing responsibilities
- **R**esources — reference material
- **A**rchive — inactive items
